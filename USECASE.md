# Sisyphus 사용 가이드

키워드와 명령어의 상세 설명 및 실제 사용 예시

---

## 키워드 (메시지 맨 앞에 작성)

### 1. `ulw` (ultrawork)

**용도**: 복잡한 작업을 병렬 에이전트와 TODO 관리로 체계적으로 수행

**동작 방식**:
- 작업을 분석하고 TODO 리스트 자동 생성
- 독립적인 작업은 병렬 에이전트로 동시 실행
- 진행 상황 추적 및 완료 검증

**사용 예시**:
```
ulw 로그인 기능 구현해줘
ulw 이 프로젝트 전체 리팩토링 해줘
ulw API 엔드포인트 5개 추가해줘
ulw 테스트 코드 작성하고 커버리지 올려줘
```

**언제 사용?**
- 여러 파일 수정이 필요한 작업
- 복잡한 기능 구현
- 단계별 진행이 필요한 작업

---

### 2. `search`

**용도**: 코드베이스에서 빠르게 파일/코드 검색

**동작 방식**:
- 병렬 Explore 에이전트로 여러 검색 동시 수행
- 파일명, 내용, 의미론적 검색 조합

**사용 예시**:
```
search 로그인 관련 파일
search API 엔드포인트 정의된 곳
search 에러 핸들링 코드
search 데이터베이스 연결 설정
search 환경변수 사용하는 파일
```

**언제 사용?**
- 특정 기능이 어디에 구현되어 있는지 모를 때
- 코드베이스 파악할 때
- 수정할 파일을 찾을 때

---

### 3. `analyze`

**용도**: 코드 심층 분석 (구조, 동작, 문제점 파악)

**동작 방식**:
- LSP 도구로 심볼 정의, 참조 추적
- AST 분석으로 코드 패턴 파악
- 의존성, 데이터 흐름 분석

**사용 예시**:
```
analyze 이 함수 뭐하는거야
analyze 이 버그 원인 찾아줘
analyze 이 클래스의 의존성 구조
analyze 성능 저하 원인 파악해줘
analyze 이 API의 호출 흐름
```

**언제 사용?**
- 코드 동작 이해가 필요할 때
- 버그 원인 파악할 때
- 리팩토링 전 영향 범위 파악할 때

---

### 4. `ultrathink`

**용도**: 복잡한 문제에 대해 깊이 사고하고 설계

**동작 방식**:
- 문제를 여러 관점에서 분석
- 장단점 비교, 트레이드오프 검토
- 최적의 접근 방식 도출

**사용 예시**:
```
ultrathink 마이크로서비스 아키텍처로 전환 어떻게 할까
ultrathink 인증 시스템 설계
ultrathink 캐싱 전략 뭐가 좋을까
ultrathink 이 레거시 코드 어떻게 개선할까
ultrathink 확장성 있는 데이터 모델 설계
```

**언제 사용?**
- 아키텍처 결정이 필요할 때
- 여러 선택지 중 고민될 때
- 장기적 영향이 큰 결정을 내릴 때

## 실전 워크플로우 예시

### 예시 1: 새 기능 구현

```
1. search 인증 관련 파일          → 기존 코드 위치 파악
2. analyze 현재 인증 구조          → 기존 구현 이해
3. ultrathink JWT vs 세션 방식     → 설계 결정
4. ulw JWT 인증으로 전환           → 실제 구현
```

### 예시 2: 버그 수정

```
1. search 에러 메시지 관련 코드    → 에러 발생 위치 찾기
2. analyze 이 함수의 호출 흐름     → 원인 분석
3. ulw 버그 수정하고 테스트 추가   → 수정 및 검증
```

### 예시 3: 코드 리뷰 준비

```
1. analyze 이번 PR 변경사항        → 변경 내용 파악
2. search 영향받는 다른 코드       → 사이드 이펙트 확인
3. ultrathink 이 설계가 적절한가   → 개선점 도출
```

### 예시 4: 레거시 코드 개선

```
1. analyze 이 모듈 전체 구조       → 현재 상태 파악
2. ultrathink 리팩토링 전략        → 접근 방식 결정
3. /plan 단계별 리팩토링 계획      → 구체적 계획 수립
4. ulw 1단계 리팩토링 실행         → 점진적 개선
```

## MCP 도구 (자동 사용)

플러그인 전체 설치 시 Claude가 필요에 따라 자동으로 사용하는 도구들:

### LSP 도구
- `lsp_goto_definition` - 심볼 정의로 이동
- `lsp_find_references` - 모든 참조 찾기
- `lsp_workspace_symbols` - 워크스페이스 심볼 검색
- `lsp_document_symbols` - 파일 내 심볼 목록
- `lsp_rename` - 심볼 이름 일괄 변경
- `lsp_hover` - 타입 정보/문서 확인
- `lsp_diagnostics` - 에러/경고 확인

### AST 도구
- `ast_search` - AST 패턴으로 코드 검색
- `ast_replace` - 패턴 기반 코드 치환
- `ast_refactor` - 내장 리팩토링 규칙 적용
- `ast_find_functions` - 모든 함수 정의 찾기
- `ast_find_classes` - 모든 클래스 정의 찾기
- `ast_find_imports` - import 문 찾기

## 팁

1. **키워드는 맨 앞에**: `ulw 작업내용` (O) / `작업내용 ulw` (X)
2. **구체적으로 요청**: "코드 고쳐줘" 보다 "로그인 실패 시 에러 메시지 표시 안되는 버그 수정해줘"
3. **단계별 접근**: 복잡한 작업은 search → analyze → ultrathink → ulw 순서로
4. **간단한 건 /quick**: 오타 수정, 변수명 변경 등은 `/quick`으로 빠르게
